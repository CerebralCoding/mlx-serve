## Building

- **First-time setup**: `./scripts/fetch-zig.sh` stages the pinned Zig nightly at `.zig-toolchain/` (homebrew's `zig` formula still ships 0.16.0, which no longer builds вЂ” see the version-gate comptime block at the top of `build.zig`). Put it on PATH (`export PATH="$PWD/.zig-toolchain:$PATH"`) or invoke `.zig-toolchain/zig` directly. CI runs the same script (`.github/workflows/release.yml`). Retire this once 0.17.0 ships stable and homebrew catches up.
- **Zig 0.17 caches configure-time subprocess output in `.zig-cache`** (`b.runAllowFail` вЂ” e.g. build.zig's `xcrun --show-sdk-path`): after a toolchain/SDK change (a macOS upgrade removing the CLT), the link keeps referencing the GHOST path until `rm -rf .zig-cache`. build.sh/xcode-build-server.sh prefer the CLT SDK but fall back to `xcode-select -p` when it's absent (story: docs/gotchas/app.md).
- **ALWAYS `zig build -Doptimize=ReleaseFast` вЂ” never bare `zig build`** (Debug is 2вЂ“4Г— slower в‡’ fake regressions; ReleaseFast binary в‰€ 7 MB, Debug в‰€ 2Г—). Rebuild the exe before any live perf A/B вЂ” `zig build test` does NOT refresh `zig-out/bin/mlx-serve`.
- Swift app: `bash app/build.sh` (see app/CLAUDE.md). The two bundle binaries (MLXCore + mlx-serve) move together.
- mlx + mlx-c are NOT brew deps: pinned submodules built by `scripts/build-mlx.sh` (deployment target 26.2 в†’ MLX's NAX kernels compiled in; script + `tests/test_mlx_staged_nax.sh` ASSERT the metallib contains `*_nax` вЂ” the CMake gate fails silently). The brew bottle is built at 26.0 and hard-wires `is_nax_available()` false even on M5. Effective min macOS is 26.2 everywhere (app `LSMinimumSystemVersion`, `Package.swift`, bundled dylibs). Bump = checkout new tag in submodule в†’ rerun script в†’ re-diff `src/mlx.zig` externs. Brew floor remaining: webp в‰Ґ 1.6.0 (`verifyBrewDeps`). mlx v0.32.0 = the runtime the M5 NAX profile was calibrated on.
- Rebuild Jinja after `lib/jinja_cpp/*.cpp` changes: `cd lib/jinja_cpp && for f in jinja_wrapper caps lexer parser runtime jinja_string value; do clang++ -std=c++17 -O2 -DNDEBUG -I . -c $f.cpp -o obj/$f.o; done && ar rcs libjinja.a obj/*.o`

## Testing вЂ” TDD is mandatory for every change

Order is fixed: (1) failing test FIRST, failing for the right reason; (2) minimum code to green; (3) full suite (`zig build test` must be 6/6 steps, 0 fail + `cd app && swift test`/`build` + the relevant `tests/*.sh`); (4) then refactor under the tripwire. A live curl is a sanity check, NOT a test.

Per change class: feature = unit test that fails without it (+ integration script if HTTP-observable). Bug fix = regression test redв†’fixв†’green, verified red-on-revert. Cross-arch work = cover every touched arch (one arch's pass в‰  the suite's pass). Refactor = characterization test first if coverage is missing. UI/build scripts = factor a pure helper and test that.

**Class bugs get class guards вЂ” no whack-a-mole.** A live failure revealing a CLASS (escaping gap, format-family leak, dispatch hole) ships THREE things: the instance regression test; a corpus entry or universal invariant in `src/format_corpus_test.zig` so future families are covered automatically; and a 1вЂ“3-line rule in `## Rules` here + full story in `docs/gotchas/` (growth policy above).

Key hermetic suites: `zig build test -Dtest-filter="format corpus"` (captured model outputs Г— split/strip/parse + no-tag-leak + valid-JSON-args + declared-type + buried-param invariants), `-Dtest-filter="tool traffic"` (replay harness). Full script matrix: `tests/CLAUDE.md`.

## Conventions

- Minimal DRY Zig; tests at the bottom of each source file; shell integration tests in `tests/`.
- Inference thread is the SOLE mlx caller (even array frees) вЂ” media gen posts to `gen_queue`, never a gpu mutex. A long gen blocks chat decode (accepted; single GPU).
- Concurrent requests batch-decode on pure-attention archs; `--max-concurrent` sizes the submit queue, not a decode gate. Slots entering a batch mid-generation drain lazy pipeline state first (`drainPipelineForBatch`).
- KV reuse via prompt-prefix matching; invalidated after tool calls + pad-only gens; hot cache spills to the SSD tier; RAM invalidation propagates to disk.
- Chat templates live in model dirs; Jinja renders with fallback formatting.

## HTTP APIs

- **OpenAI chat/completions + Responses**: chat usage ALWAYS carries `prompt_tokens_details.cached_tokens` (0 when cold; ONE `formatChatUsage` helper across non-stream/tool-call/stream-final вЂ” black-box tooling like llmprobe reads only this field, so an unreported cache scores as "no prompt caching"); chat thinking opt-ins = standard `reasoning_effort` string (none=off; effortв†’budget via shared `responses_mod.effortBudget`; unknown efforts enable at default budget) OR vendor `enable_thinking` bool вЂ” either enables; explicit `reasoning_budget_tokens` outranks the effort budget. `n>1` choices honestly 400 on chat+completions (`nChoicesRejectReason` вЂ” single-choice engine, silent one-choice no-op is the llmprobe-flagged class). `/v1/responses` envelope echoes request fields; every SSE event gets `sequence_number` (injected by `sendResponsesEvent`); `background:true` в†’ honest 400; stateful chains via `ResponseStore`/`previous_response_id` (missing в†’ 404, function_call_output в†’ final-answer mode); compaction is pure data (no LLM call); WS transport = same endpoint via Upgrade, NO `[DONE]` on WS success (`[DONE]` is HTTP-SSE-only; a WS `[DONE]` breaks chained sessions).
- **Anthropic `/v1/messages`** (Claude Code): top-level `system` в†’ system msg; typed blocks (`tool_result`в†’role tool, `thinking`в†’enable_thinking+budget, fake `signature`); `input_schema`в†’`parameters`; stop-reason map stop/length/tool_calls в†’ end_turn/max_tokens/tool_use, a matched client stop sequence в†’ `stop_sequence` + the sequence echoed (never masks max_tokens/tool_use вЂ” `anthropicStopReason`); full SSE content-block lifecycle. Launcher env: `ANTHROPIC_BASE_URL` + dummy keys + `ANTHROPIC_DEFAULT_*_MODEL=mlx-serve` + `CLAUDE_CODE_SUBAGENT_MODEL`.
- **Ollama `/api/*`**: pure translation (ollama.zig), NO duplicated inference вЂ” request в†’ OpenAI body (`num_predict`в†’max_tokens, imagesв†’data URLs, `format`в†’response_format, `think`в†’enable_thinking, stream defaults TRUE), response re-framed by `Sink` on `Conn.ollama_sink` (SSEв†’NDJSON, args stringв†’object, `reasoning_content`в†’thinking, usageв†’eval counts; glue clears `stream.ollama_sink` before `sink.finish()`). No-model endpoints answered pre-scheduler (never cold-load); `resolveName` handles `name:tag`. Discovery is two-level (`org/model`); one path must never register under TWO ids (`registry.peekByPath` вЂ” double-residency OOM class).
- **LAN sharing**: proxy is a TRANSPORT (no scheduler/MLX involvement); keyless-mode gate = `routeClass` allowlist Г— `SharedSet` (everything else non-loopback в†’ 403; with `--api-key` set, key semantics rule unchanged); `<id>@<peer>` mirroring with `lan_peer` badge; byte-for-byte tunnel rewriting only the model field (decide+rewrite share ONE parsed slice); loops impossible by construction (TXT self-token, remote entries never re-exported, `@peer` denied non-loopback); load/unload on remote ids = 200 no-ops; distinct failure modes incl. a 15 s rediscovery wait. Full design: `docs/reference.md`.
- **Observability** (`--metrics`): zero cost when off (one null-check per request at `finishSlot`, never per token); `vllm:` + `mlx_serve:` names; TTFT captured exactly at prefill completion; live tok/s via ONE atomic published per decode tick (`inflight_generated_tokens`; excludes finished slots so live==total at rest); prefill liveness gauges (`prefill_tokens_live` per chunk + `requests_prefilling` at entry). `--api-key`: loopback TRUSTED and exempt; Bearer/x-api-key/Basic/`?api_key=`; `/health`+OPTIONS open; `constTimeEql`. No admin surface (deliberate). Panel/rate rules in ## Rules.

## Tool calling (server pipeline)

With `tools` present, tokens buffer for detection (`<tool_call>`, Hermes XML, Gemma `<|tool_call>` incl. double-brace unwrap, hy3 suffixed tags, DSV4 wrappers, raw JSON); thinking buffered separately. Parse chain: strict в†’ tolerant repairs в†’ truncation salvage; then the per-request chokepoint `server.parseToolCallsForRequest` = parse в†’ inferred-name filter в†’ parallel clamp (`parallel_tool_calls:false` / Anthropic `tool_choice.disable_parallel_tool_use` keep ONLY the first parsed call вЂ” client-requested, so NOT autocorrect-gated; model re-issues the rest next round) в†’ buried-param hoist в†’ schema coercion (hoist+coercion gated by `--no-tool-autocorrect`; a final safety net guarantees emitted `arguments` are ALWAYS valid JSON). Serialization (`chat.serializeMessagesJson`): role "tool" passed natively (Gemma renders `<|turn>tool` вЂ” no `tool_responses` field, causes duplicates), args as JSON STRINGS, every arbitrary-byte string through `appendJsonString`. Streaming: full args in ONE SSE delta; thinking buffered until close, emitted as `reasoning_content`. Fallback (`fallbackFormatChat`): ChatML, Llama ipython, Gemma "Tool result:". Client side: `app/CLAUDE.md`.

## Rules (distilled gotchas вЂ” full stories in docs/gotchas/)

### Tool calling & formats (в†’ docs/gotchas/tool-calling.md)

- **Control bytes**: ONE raw byte <0x20 anywhere in history kills the strict nlohmann render в†’ SILENT downgrade to `fallbackFormatChat` в†’ wrong-family tags, model degenerates (loses its stop token). Every arbitrary-byte string goes through `appendJsonString`; render failure logs WARN. Wrong-family tags in output в‡’ suspect silent fallback before suspecting the model.
- **Model-mangled arg JSON**: after strict parse fails, `looseRepairToolCallJson` re-escapes ctl bytes / inner quotes / invalid backslashes, re-validated by strict re-parse; never drop the whole call (it leaks into content and wastes the turn). Model-agnostic; helps 1B models, leaves valid JSON untouched.
- **Truncated opener** (`close_rel==null`): try JSON shapes, then Hermes, then XML on the remainder; recover NAME + `{}` вЂ” NEVER ship partial content values (a half-written file is worse than a re-issued chunked write).
- **Delimiter-drop tolerance (hy3 class)**: a tag parser never bails a whole call on ONE missing/mismatched delimiter вЂ” name ends at the earliest structural marker, key-closes match tolerantly, plural-wrapper recovery keys strictly on the SUFFIXED `<tool_calls:` form (bare `<tool_calls>` belongs to DSV4).
- **Loop-stop cuts are truncations**: degenerate-tail cut в‡’ `finish_reason "length"` (`scheduler.loopStopReason`), logged `[loop-stop]`, and the Gemma truncation salvage ships NO fragment values (unterminated pairs dropped via `cut`; completed pairs survive). A tool-executor error must never infer "truncated" вЂ” only the loop knows `maxTokensHit`.
- **Types come from the SCHEMA, never the value's spelling**: `coerceToolArgsToSchema` coerces scalars BOTH ways ("False"в†’bool; literal-spelling strings stay strings); undecidable в†’ untouched so client errors stay honest. ONE chokepoint `server.parseToolCallsForRequest` вЂ” never call `chat.parseToolCalls` directly from a handler (silent per-surface regression; output-equality tests can't see it).
- **Buried required params**: `hoistMisplacedRequiredParams` lifts a required top-level scalar buried in a container ONLY on all-schema-read conditions (required+absent at top, scalar, container items don't declare it, unanimous same typed value); ambiguity в†’ untouched. The parse layer is innocent in this class вЂ” pristine correctly-escaped args are the tell.

### Server, HTTP, lifecycle (в†’ docs/gotchas/server-http.md)

- **Liveness is a property of the SOCKET, never the generator**: every streaming token loop calls `beatStreamKeepalive` at the BOTTOM of each iteration (or before an early `continue`); emits only when no bytes for 5 s (`Conn.heartbeat` stamped at the write sites). KNOWN GAP: Ollama sink drops SSE comments (fix if reported: empty-content NDJSON line).
- **Teardown with detached workers**: `serve` drains conn threads (atomic `active_conn_threads` + `cancelAllInFlight`; `waitNext` returns `.done` on cancel) BEFORE `scheduler.deinit`. A `defer deinit()` after a spawn loop with dropped handles is the classic use-after-free shape.
- **Non-text model on a text surface must 400 BEFORE prefill** (gen-stub tokenizer в†’ null transformer deref = one-request server kill): `textGenRejectReason` at ALL text-gen surfaces, TWICE (pre-ensureLoaded peek via discovery `arch_hint` вЂ” don't cold-load 15 GB to earn a 400 вЂ” and post-load authoritative). New surface в†’ extend `isTextGenRoute`; new modality в†’ engine slot + `modalityFromType`; same taxonomy feeds `mlx-serve run` preflight + `list` TYPE column.
- **Auto-context is PINNED at load** (`pinAutoContext`) вЂ” clients bake `meta.context_length` into config files once; per-request recompute drifts with system load. The 85% margin applies to the MEMORY ceiling ONLY, never the model max. Never gate context behavior on `server_config.max_context_size` (set only by `--ctx-size`) вЂ” ask `getEffectiveContextLength`. App budgets (`AgentBudget` в†’ pi/opencode/Claude Code configs) derive from the advertised value verbatim вЂ” no hardcodes, no double margin.
- **GPU memory ceiling must see EXTERNAL pressure**: `currentGpuMemoryCeiling = min(getGpuWorkingSetLimit, mlx_active+cache+free)`; reserve the full prefix-cache budget up front. Metal OOM is UNCATCHABLE (async GCD `std::terminate`) вЂ” prevention only; its backtrace shows libllama frames even for pure-MLX (terminate handler вЂ” red herring). Embedded-engine (ds4/llama) requests are EXEMPT from MLX memory guards (`mlxMemoryGuardApplies` вЂ” stub dims bill a transformer that never runs); any new MLX guard skips them or routes through `checkAttentionMemory`.
- **`--timeout` is a STALL timeout** (seconds without a NEW token, `generate.StallClock`) вЂ” a producing request never times out. `toolCallFinishReason` preserves "length" through tool parse on all four surfaces (clients' truncation recovery must fire; "tool_calls" on a server-cut fragment burned 15 min of GPU). `outputBudgetGuidance` only when tight (<12288) вЂ” a huge honest number overrides the chunking convention and invites mega-calls.
- **Ownership by PROVENANCE, never content** (sentinel-by-content leak class): a `catch "<literal>"` fallback whose defer frees on `!mem.eql(result, literal)` leaks when the literal is also a LEGAL payload (escaping an empty string yields the same `""` as the OOM fallback вЂ” every all-reasoning/empty-content response leaked). Use `jsonEscapeOrEmpty`-style `{slice, owned}` returns; the debug-allocator `leaked` lines in the integration server log are the tell.
- **LAN robustness**: dns_sd callbacks fire per-INTERFACE вЂ” complete on SUCCESS + MoreComing clear, collect ALL A records, fetch LOOPBACK-FIRST (macOS Local Network privacy silently blackholes unapproved LAN SYNs: connect poll timeout + SO_ERROR=0 is the signature); transient resolve failures retry via the `known` registry (browse never re-announces a live service) and NEVER evict an installed peer directly вЂ” removal is `attemptKnown`'s decision only, after `PEER_DROP_FAILS` consecutive failures (one mDNS hiccup вЂ” interface churn from a VM/docker bridge, busy daemon вЂ” otherwise 404s a live peer "offline" mid-chat); dead dns_sd refs (browse AND advertise) are REVIVED by the browser loop, never left null forever (an mDNSResponder restart otherwise silently ends discovery/advertising for the process lifetime). Any id compared from a raw body scan canonicalizes JSON `\/` first (`unescapeJsonSlashes` вЂ” third bite of that class; Swift clients escape every slash). Proxy failure modes stay DISTINCT (discovery-off / model-unlisted / peer-unknown в†’ poked 15 s rediscovery wait); never a silent local-default answer.

### Engine: KV, spec-decode, kernels, MLX (в†’ docs/gotchas/engine-mlx.md)

- **KV invalidation**: after tool calls (generated call tokens aren't in `cached_prompt_ids` вЂ” reuse corrupts attention) and pad-only gens. Sliding-window layers keep the full buffer (no trimming); decode views slice to last `sw`; mask handles scope.
- **Hot-cache restore ALWAYS clamps**: unconditional `truncate(final_len)` вЂ” restored offset = MATCHED length, never the snapshot's stored length (PLD leaves stale KV tails past the committed step; drift в‡’ wrong RoPE everywhere + `broadcast_shapes` mask crash on Gemma sliding layers at long ctx).

- **SSM/hybrid**: init checks `ssm.ssm_state.ctx == null`, NOT `!initialized` (conv sets initialized before recurrence state exists). Param-free RMS norm passes `ones()` (mlx-c crashes on null weight). Nemotron dt clipping: only the `time_step_limit` array, not min/max. PLD-on-hybrid snapshot/restore: per-FIELD null guards (conv without ssm_state aborts mlx-c).
- **Spec verify invariant** (PLD/drafter/MTP shared): `cache.step = prompt_len + emitted`, t1 NOT in cache on entry; verify input `[t1, draftвЂ¦]`; partial-accept correction sampled from the ORIGINAL `verify_logits[accepted]` (index `accepted`, not `accepted-1` вЂ” off-by-one silently corrupts; pinned by equivalence scripts).
- **Spec dispatch discipline**: all four HTTP surfaces Г— streaming + non-streaming wire `use_pld/use_drafter/enable_mtp`; per-request MTP default = `server.defaultEnableMtp` ONLY (MoE targets default OFF вЂ” verify pays expert routing; `--mtp` flips; a MoE head is otherwise unreachable from clients that send nothing).
